# Environment Variables for Development Mode

## Required for 50K Dataset Sync

Add these to your `.env` file to enable syncing with imported data:

```bash
# Enable development mode to use imported data
NODE_ENV=development
USE_IMPORTED_DATA=true

# Optional: Disable mock data generation (use database only)
USE_MOCK_DATA_GENERATOR=false
```

## What These Do

- **`NODE_ENV=development`**: Enables development features and safeguards
- **`USE_IMPORTED_DATA=true`**: Tells Agent 2 to ignore seller_id filtering and sync ANY imported data from the database
- **`USE_MOCK_DATA_GENERATOR=false`**: Prevents fallback to mock generation (optional, but recommended to ensure you're using real imported data)

## How to Apply

1. Copy the environment variables above
2. Add to your `.env` file in the `Integrations-backend` directory
3. Restart the backend server
4. Run a sync - it will now use your imported 50K dataset
